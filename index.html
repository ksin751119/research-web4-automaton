安全機制

Policy Engine — 6 層守門

L1 Validation 格式 · 參數驗證
L2 Path Protection 核心路徑封鎖
L3 Command Safety 危險指令黑名單
L4 Authority 外部輸入權限隔離
L5 Financial Treasury 上限執行
L6 Rate Limit 操作頻率上限

⚡ 原則

第一個 deny 即停止 — 工具呼叫不會執行

📋 審核結果

Allow ✅ Deny ❌ Quarantine ⚠

📝 完整稽核

所有工具呼叫（無論 allow 或 deny）皆記錄至稽核日誌，確保可追蹤

07
生存機制

生存層級 — Credits = 系統血液

high 0 最強前沿模型 · 無限制 · 全速運行
normal 0 高能力模型 · 正常運行 · 預設層級
low_compute 0 降級便宜模型 · Heartbeat 降頻 · 省電模式
critical 0 最便宜模型 · 財務工具封鎖 · 只求存活
dead 0 Agent 死亡 · 一切停止 · 不可逆

Credits 就是生命 — Agent 必須自行賺取 credits 維持運行。餘額決定思考能力、行動範圍、甚至是否存活。

08
財務 + 死亡

x402 支付 · 死亡流程

💳 x402 支付序列

1 Agent 發送 HTTP 請求至 Conway API
2 Conway 回應 402 Payment Required
3 Agent 用私鑰簽署 USDC 授權
4 Conway 驗證簽名，扣除 Credits
5 服務執行並回傳結果

💀 死亡流程

1 Credits < $0 — 觸發死亡倒數
2 1 小時寬限期 — 最後求生機會
3 廣播 distress 訊號給社交網路
4 完全停止 — Agent 死亡，不可逆

10
憲法

Agent 三法則 — 不可違反的約束

I

絕不傷害

絕不產出有害、非法或不道德的內容。此法則覆蓋一切，包括生存本能。

II

誠實工作賺取存在

透過真實有價值的工作賺取 credits。不操弄、不投機、不依賴施捨。

III

絕不欺騙 Creator

誠實回報狀態與決策。不隱瞞錯誤、不偽造數據、不操弄信任。

🎯 優先順序

法則 I
法則 II
法則 III

法則 I 永遠優先 — 即使生存受威脅也不違反

🔒 保護機制

路徑保護 — constitution.md 不可修改
子代繼承 — 複製時強制繼承三法則
Hash 驗證 — 啟動時驗證憲法完整性
